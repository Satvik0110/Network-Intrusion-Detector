Abstract

Network intrusion detection systems (NIDS) are critical components of cybersecurity infrastructure, designed to identify malicious activities in network traffic. In this project, we develop a comprehensive NIDS using various machine learning algorithms on the well-established KDD Cup'99 and NSL-KDD datasets. We implement and compare multiple classification techniques including Naive Bayes, Artificial Neural Networks (ANN), Decision Trees, Random Forest, Logistic Regression, Support Vector Machines (SVM), and K-Nearest Neighbors (KNN). Our approach involves both using scikit-learn's implementations and developing algorithms from scratch to provide deeper insights into their workings. Experimental results demonstrate that ensemble methods like Random Forest achieve the highest accuracy while maintaining reasonable computational efficiency. This work contributes to the understanding of how different machine learning paradigms perform in intrusion detection tasks and provides a foundation for developing more robust security systems.

Introduction

With the increasing sophistication of cyber attacks, developing effective intrusion detection systems has become paramount for network security. Machine learning techniques offer powerful tools for identifying patterns associated with network intrusions, potentially outperforming traditional rule-based systems. Our project focuses on evaluating and comparing various machine learning approaches for network intrusion detection.

We utilize two benchmark datasets in this domain:

  • KDD Cup'99: A widely-used dataset containing a wide variety of intrusions simulated in a military network environment.
  • NSL-KDD: An improved version of KDD Cup'99, addressing issues such as redundant records and improving the distribution of attack types.

By implementing both scikit-learn based and from-scratch versions of our algorithms, we aim to provide:

  • Comprehensive performance comparisons across different machine learning paradigms
  • Insights into algorithmic behavior and hyperparameter sensitivity
  • Educational value by demonstrating core algorithm implementations
  • Analysis of model suitability for real-time intrusion detection applications

Methodology

Our approach combines rigorous data preprocessing, and the implementation of seven distinct classification algorithms, both using scikit-learn and developed from scratch. We have evaluated our models through performance metrics like accuracy, precision, recall, F1-score, confusion matrices, and the ROC curve.

Data Preprocessing

We applied the following preprocessing steps to the KDD Cup'99 dataset:

  • Assigned appropriate column names for easier interpretation
  • Dropped the unnecessary columns, which are constant or uninformative
  • Converted categorical columns (protocol_type, service, flag) to numerical format using one-hot encoding
  • Mapped the label column to binary values: 0 for normal traffic, 1 for attacks
  • Normalized all numerical features using standardization (zero mean and unit variance)
  • Split the dataset into training and testing sets with an 80-20 ratio for KDD Cup'99 and used the Train and Test files for NSL-KDD Dataset

Classification Models

Naive Bayes

Probabilistic classifier based on applying Bayes' theorem with strong independence assumptions between features.

Artificial Neural Networks

Multi-layer perceptron architecture with backpropagation for learning complex patterns in network traffic.

Decision Tree

Tree-based model that creates a flowchart-like structure to make decisions based on feature values.

Random Forest

Ensemble method combining multiple decision trees to improve accuracy and control overfitting.

Logistic Regression

Statistical model that applies a logistic function to predict the probability of an intrusion event.

Support Vector Machines

Creates an optimal hyperplane in high-dimensional space to separate normal from intrusive traffic.

K-Nearest Neighbors

Instance-based learning where classification is based on the majority class of k nearest data points.

XGBoost

XGBoost builds an ensemble of decision trees using gradient boosting to optimize prediction accuracy.

Implementation Approach

For each algorithm, we created two implementations:

  • Scikit-learn Based: Utilizing the optimized implementations from the scikit-learn library with hyperparameter tuning.
  • From Scratch: Developing custom implementations to gain deeper insights into algorithm behavior and adaptability.

Evaluation Metrics

We evaluated each model using the following metrics:

  • Accuracy
  • Precision, Recall, and F1-score
  • False Positive Rate and False Negative Rate
  • Area Under ROC Curve (AUC)
  • Training and Inference Time

Key Findings

  • Random Forest (scikit-learn) achieved the highest accuracy and F1-Score on the KDD Cup'99 dataset (99.99% accuracy, 0.9998 F1), making it the top-performing algorithm on that dataset.
  • SVM (from scratch) recorded the highest accuracy (90.09%) and F1-Score (0.9117) on the NSL-KDD dataset, outperforming even the scikit-learn implementation of SVM and other models on this dataset.
  • Decision Trees demonstrated strong performance across both datasets. On KDD Cup'99, the scikit-learn version scored 99.89% accuracy with 0.9987 F1, and on NSL-KDD it maintained solid performance (82.23% accuracy, 0.8199 F1).
  • Naive Bayes was consistent across implementations on the KDD Cup'99 dataset (97.37% accuracy, 0.9665 F1), but performance dropped significantly on NSL-KDD, especially in the from-scratch version.
  • KNN (scikit-learn) showed good results on KDD Cup'99 (99.81% accuracy, 0.9976 F1), but dropped on NSL-KDD (78.87% accuracy, 0.7769 F1).
  • ANN achieved strong results on KDD Cup'99 using scikit-learn (99.70% accuracy, 0.9963 F1), but had relatively lower performance on NSL-KDD (79.98% accuracy, 0.7909 F1).
  • Overall, the NSL-KDD dataset posed a greater challenge than KDD Cup'99, as it addresses key issues such as redundant records and class imbalance present in the original dataset. By providing a more balanced and realistic distribution of attack and normal traffic, NSL-KDD better tests the generalization capability of algorithms, leading to lower but more meaningful performance metrics.
